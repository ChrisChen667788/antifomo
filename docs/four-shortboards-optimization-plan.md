# Anti-FOMO 四条短板优化方案

更新时间：2026-03-26

## 短板 1：公众号 URL 获取仍依赖 UI 自动化

### 问题

- 没有官方个人公众号 URL API
- 文章经常开在微信内嵌页，不能直接读浏览器地址
- 现有主路径虽然已经是 URL-first，但稳定性还受微信桌面端 UI 变化影响

### 优化方案

1. `URL-first` 固化为正式主路径
   - 浏览器 URL
   - 微信文章页分享/复制链接
   - 历史真链恢复
   - OCR 最后兜底
2. 为“分享/复制链接”引入 `profile`
   - `auto / compact / standard / wide / manual`
3. 增加 URL 校验
   - 域名校验
   - 标题重合度校验
   - HTML 可读性校验
4. 将 `URL 入队 / OCR 兜底` 统计前台可视化

### 当前已落地

- `article_link_profile` 已进入后端和 Web `Collector` 配置
- 真链校验已加入 URL 入队前

## 短板 2：研报高质量官方源占比不够

### 问题

- 搜索结果容易偏媒体和聚合源
- 部分主题下官方源占比过低，导致结论可信度不够

### 优化方案

1. 在 hit 选择阶段增加官方源配额
2. 当官方源覆盖过低时，强制触发二次扩搜
3. 扩搜时优先官方站点 query
4. 在诊断面板里显式展示：
   - 官方源占比
   - 官方源不足时是否触发扩搜

### 当前已落地

- hit 选择已改成 `official -> aggregate -> media` 配额优先
- 当官方源占比/数量过低时会触发扩搜

## 短板 3：甲方/竞品/伙伴实体匹配与公开联系方式命中率不足

### 问题

- 容易把媒体标题或泛主题短语误当成甲方/竞品
- 联系方式里偶尔出现图片地址、媒体域名主页等低价值结果

### 优化方案

1. 实体候选增加角色过滤
   - 甲方更偏官方/业主/预算/项目语境
   - 伙伴更偏咨询/渠道/集成/牵线语境
   - 竞品更偏中标/平台/交付/方案语境
2. 在媒体源上提高实体入选门槛
3. 联系方式仅保留：
   - 公开邮箱
   - 公开电话
   - 官方/公开联系页
   - 采购公告联系人
4. 屏蔽：
   - 图片文件名伪邮箱
   - 媒体/社区站点根域名
   - 无业务价值的公开页

### 当前已落地

- `target` 与 `partner` 的媒体源实体门槛已提高
- 联系方式提取已过滤图片 URL 和泛媒体主页

## 短板 4：研报 UI 和证据工作台成熟度还不足

### 问题

- 生成、历史、专题、总结页之间的研报结构还没完全统一
- 用户需要更明确地看到：
  - 为什么这份研报可信
  - 为什么给出这些甲方/竞品/伙伴
  - 当前用了哪些信息源

### 优化方案

1. 统一四处的研报结构：
   - 生成结果
   - 研报中心列表
   - 专题页
   - Session Summary
2. 统一诊断面板字段：
   - 检索质量
   - 严格主题命中
   - 官方源占比
   - 域名覆盖
   - 实体归一化数量
3. 统一 Top 3 评分拆解与证据入口
4. 强化专题页 diff 和版本工作台

### 当前已落地

- `retrieval evaluator + 主题收敛 + 实体图谱` 已在 Web/小程序同步展示
- `Top 3 评分拆解`、来源可信度分层、专题 diff 已有基础版本

## 推荐开发顺序

1. 继续稳定 URL-first 主路径
2. 提高官方源占比与官方源触发扩搜成功率
3. 继续强化联系方式和实体命中
4. 最后做研报工作台 UI 的专业化收口
