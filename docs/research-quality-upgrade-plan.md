# 解决方案智囊研报质量升级方案

## 本轮采用的方法

- 咨询备忘录结构：采用 `结论 -> 证据 -> 动作` 的三段式执行摘要，并把标题收敛到 `区域｜场景/赛道｜公司/客户类型：预算窗口与进入路径研判`。
- 问题树式范围规划：在检索前先锁定 `区域 / 行业 / 场景 / 公司 / 排除项`，再按 `甲方 / 预算 / 招标 / 伙伴 / 竞品` 扩展查询。
- Corrective Retrieval 思路：当首轮召回弱时，继续做定向扩搜和公司官网/IR/联系页补源，而不是直接让模型在弱证据上成稿。
- 实体图谱归一化：将甲方、竞品、伙伴、公开联系方式、活跃团队拆成结构化维度，优先用归一化结果纠偏标题和摘要。

## 对应到当前系统的实现

- `gpt-5.4`
  - 前置策略规划：范围收敛、排除项、定向 query expansion
  - 后置策略纠偏：标题、执行摘要、咨询视角二次裁决
- `claude-opus-4-6`
  - 主成稿：结构化研报、章节展开、行动卡
- 工程规则
  - 主题硬约束
  - 区域冲突过滤
  - 伪实体清洗
  - 公开联系方式与活跃团队抽取
  - 低证据兜底

## 下一轮继续优化的 4 个点

1. 标题压缩与词元切割
- 继续收紧窄主题的场景词优先级，减少“泛 AI / 泛数字化”词元上浮。
- 对标题做长度和噪声词约束，优先保留区域、场景、主体、阶段。

2. 纠错检索
- 在 `retrieval_quality = low` 时追加一轮官方源与公司 seed 的矫正检索。
- 提高官方源占比和有效域名覆盖。

3. 联系方式与活跃团队
- 扩充已知公司官网/IR/contact seed。
- 对公开联系人、服务热线、采购联系人、区域团队线索继续做规则增强。

4. UI 诊断透明度
- 前台明确展示“当前标题/摘要来自主成稿还是纠偏后版本”。
- 让用户看清这份研报的证据强度、范围锁定和后续验证路径。
